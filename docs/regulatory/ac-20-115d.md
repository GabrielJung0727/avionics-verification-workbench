# FAA AC 20-115D — Airborne Software Development Assurance

## Scope (1 line)
FAA Advisory Circular that **accepts RTCA DO-178C** (and its supplements
DO-330 / DO-331 / DO-332 / DO-333) as a means of compliance for civil
airborne software.

## What it says (in plain words)
- DO-178C is recognised as an acceptable means
- DO-330 covers tool qualification when tools claim certification credit
- The supplements (model-based, OO, formal methods) extend DO-178C in
  specific contexts
- Industry data and Order 8110.49A inform software approval

## What is intentionally out of scope
- DO-331 (model-based) deliverables — none of our schemas are derived
  from a certified modelling tool
- DO-332 (object-oriented) — code is procedural / dataclass-based
- DO-333 (formal methods) — no formal proofs claimed
