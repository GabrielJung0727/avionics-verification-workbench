# DO-254 — Design Assurance Guidance for Airborne Electronic Hardware

## Scope (1 line)
Hardware analogue of DO-178C: design assurance for **airborne electronic
hardware** — FPGAs, ASICs, complex PLDs, custom boards.

## Status in this repo: **OUT OF SCOPE — explicit declaration**

The workbench has no airborne electronic hardware. The HIL-lite
"loopback MCU" is a *software surrogate* whose intended use is to
exercise the runtime assurance shell, not to substitute for a real
hardware design assurance activity.

## What is NOT done here
- Hardware requirements capture under DO-254
- Hardware verification (simulation, in-circuit, integration test)
- HDL coding standards & reviews
- Hardware life-cycle data (HLD, HDD, HCMP, HCI)
- FAA acceptance via AC 20-152A

## If this project were extended to real hardware
- Promote `tools/sim_py/avx_sim/hil/` from loopback to a physical MCU
  bring-up (Phase D mentions UART/UDP-extensible)
- A real HDL deliverable would need its own `docs/regulatory/do254-*.md`
  set + a DO-330-qualified HDL toolchain
