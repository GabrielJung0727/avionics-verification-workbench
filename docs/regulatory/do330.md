# DO-330 — Software Tool Qualification Considerations

## Scope (1 line)
Process for qualifying a **tool** when its output is used to claim
certification credit for airborne software (DO-178C / DO-278A).

## What it covers
- Tool **use criteria**:
  - **Crit 1**: tool output becomes part of airborne software
  - **Crit 2**: tool *automates* a verification activity AND its output is used to claim credit
  - **Crit 3**: tool reduces / eliminates a process AND its output is used to claim credit
- Tool Qualification Levels (**TQL-1 .. TQL-5**) derived from criterion + DAL
- Tool operational requirements
- Tool verification and validation activities
- Tool quality assurance

## What is intentionally out of scope
- An actual Tool Qualification Plan (TQP)
- An actual Tool Qualification Accomplishment Summary (TQAS)
- Independence between tool development and tool verification (single engineer)
- Substantiated TQL — values shown are hypotheses

## How TQL maps to use criterion + DAL
| Use criterion | DAL A | DAL B | DAL C | DAL D/E |
|---|---|---|---|---|
| 1 | TQL-1 | TQL-2 | TQL-3 | TQL-4 |
| 2 | TQL-4 | TQL-5 | TQL-5 | TQL-5 |
| 3 | TQL-5 | TQL-5 | TQL-5 | TQL-5 |

Most repo tools land at **Crit 2 / 3 with hypothetical DAL B** → **TQL-5**.
